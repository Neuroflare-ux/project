Incident Response Playbooks

Step-by-step guides for common cloud security incidents.

  1. Immediately revoke the affected access keys or tokens in your cloud console.
  2. Check CloudTrail / audit logs for any actions taken with those credentials.
  3. Identify what resources were accessed or modified.
  4. Rotate all secrets and API keys across affected services.
  5. Enable MFA on the account if not already active.
  6. Document the incident and notify your security team.

  1. Set the bucket ACL to private immediately in the cloud console.
  2. Remove any public access block exceptions.
  3. Review bucket contents — identify what data was exposed.
  4. Check access logs to see if any external IPs accessed files.
  5. Notify affected users if personal data was involved.
  6. Enable bucket versioning and access logging going forward.

  3. Restore from the most recent clean backup.
  4. Identify the entry point (phishing, RDP, unpatched software).
  5. Patch the vulnerability before bringing systems back online.

  3. Block offending IP ranges using firewall/security group rules.
  4. Scale up resources temporarily to absorb traffic if needed.
