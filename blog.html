Breach Case Studies
Case Study
The Capital One Breach

July 2019 | AWS

A misconfigured AWS WAF allowed an SSRF attack, giving access to the EC2 metadata service and temporary IAM credentials. Over 100 million customer records were exposed.

Lesson: Restrict EC2 metadata access using IMDSv2. Audit WAF rules regularly.

Case Study
Uber's 2022 Social Engineering Attack

September 2022 | Social Engineering

An attacker posed as IT support and convinced an employee to approve an MFA push. They then accessed Slack, HackerOne reports, and cloud infrastructure — no malware used.

Lesson: MFA alone is not enough. Verify identity through a second channel.

Case Study
SolarWinds Supply Chain Attack

December 2020 | Supply Chain

Attackers compromised SolarWinds' build pipeline and inserted malicious code into a legitimate update downloaded by 18,000 organisations including US government agencies.

Lesson: Verify third-party software integrity. Scan dependencies with Trivy and Gitleaks.

Case Study
MOVEit Transfer Mass Exploitation

May 2023 | Zero-Day

The Cl0p group exploited a SQL injection zero-day in MOVEit Transfer. Over 2,700 organisations were affected including government agencies, airlines, and banks.

Lesson: Patch file transfer tools immediately. Monitor for unusual outbound transfers.

Guide
Why S3 Buckets Get Exposed

AWS Storage

Public S3 buckets happen when developers set them to public during testing and forget to revert before production. Enable Block All Public Access at the account level, not just per bucket.

Guide
IAM Least Privilege — What It Actually Means

Identity & Access

Least privilege means giving each service or user only the exact permissions they need. Start by auditing any role with *:* permissions and removing admin access from service accounts.

Container Security Basics for Cloud Teams

Docker & Kubernetes

CVE-2024-21626 showed a single container escape can give host access. Run containers as non-root, use read-only filesystems, and scan images before deployment with Trivy.

How Attackers Exploit CI/CD Pipelines

DevOps Security

CVE-2024-23897 and CVE-2024-27198 allowed unauthenticated access to Jenkins and TeamCity. Never expose CI/CD dashboards publicly. Use VPN or SSO and keep tools patched.

Fundamentals

A CVE is a public identifier for a known security flaw scored from 0–10. A CVSS score of 9+ means patch immediately. Use the Security Assessment page to see which CVEs affect your stack.

Vulnerability vs Misconfiguration

Fundamentals

A vulnerability is a flaw in code. A misconfiguration is a correct feature set up incorrectly — like a public S3 bucket. Misconfigurations cause 43% of cloud breaches and are often easier to fix.
